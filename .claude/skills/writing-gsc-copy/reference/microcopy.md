# Microcopy patterns

Short, high-frequency UI strings. Consistency here is what makes a product feel
built by one hand.

## Contents
- Buttons & CTAs
- Form labels, placeholders, help
- Validation & errors
- Empty states
- Confirmations & success
- Email

## Buttons & CTAs

Name the action; verb + object. Never "Submit", "Learn more", "Click here".

| Good | Bad |
|---|---|
| Request the data room | Submit |
| See the thesis | Learn more |
| Request access | Get started |
| Download the memo | Click here |
| Talk to us | Contact |

Primary button per view: one. Everything else is secondary/ghost.

## Form labels, placeholders, help

- **Label:** always present, short, human. "Work email", not "Email address (required)".
- **Placeholder:** an example, never the label. "jane@fund.com".
- **Help text:** only when it prevents an error. "We'll only use this to send the data-room link."
- Ask for the minimum. Every field is friction.

## Validation & errors

Calm, specific, actionable. State what happened + what to do. No codes, no blame.

| Good | Bad |
|---|---|
| That email didn't go through. Check the address and try again. | Error 400: invalid input |
| Enter a work email so we can verify your firm. | Email is required! |
| We couldn't reach the server. Try again in a moment. | Unknown error occurred |

## Empty states

Explain what goes here + the first action. Never a dead end.
> **No holdings yet.** Once a club closes, it appears here. **Browse opportunities →**

## Confirmations & success

Understated. Confirm + set expectation. No confetti.
> **You're on the list.** We review requests within two business days.

## Email

- **Subject:** specific, no clickbait. "Comunicaciones FC — investment memo" beats "You won't believe this opportunity".
- **Body:** lead with why you're writing, then the one action. Sign as a person.
- **Footer:** the risk line + a real unsubscribe. Confidential where applicable.
